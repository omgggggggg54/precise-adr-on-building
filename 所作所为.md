# 所作所为

## 本次目标
1. 在当前 PreciseADR 项目中引入药物分子结构编码特征。  
2. 引入时序建模能力（利用 `receipt_date`）。  
3. 保持原训练流程可用，尽量少改动现有主干逻辑。

## 我做了什么

### 1) 新增分子结构编码模块
- 新增文件：`dataset/mol_features.py`
- 提供 4 种编码模式：
  - `atom_bond`：原子/化学键信息哈希聚合
  - `fingerprint`：Morgan 指纹（无 RDKit 时自动回退 n-gram 哈希）
  - `molformer`：优先读取外部 MolFormer 向量表，缺失时回退 `hybrid`
  - `hybrid`：`atom_bond` 与 `fingerprint` 融合
- 支持读取 `DrugBank ID -> SMILES` 的 CSV。
- 输出统一为定长 `drug_struct_dim` 向量。

### 2) 在数据构图阶段接入药物结构特征
- 修改文件：`dataset/please.py`
- 主要改动：
  - `PLEASESource` 新增 `args` 参数读取结构编码配置。
  - `build_base_graph()` 中对每个 drug 节点构建 `struct_feat`。
  - 构建完成后打印关键统计（命中 SMILES 数、MolFormer 命中数、是否启用 RDKit）。
  - `processed_file_names` 升级到 `v3` 并带上结构配置，避免旧缓存混用。

### 3) 在数据构图阶段接入时间特征
- 修改文件：`dataset/please.py`
- 主要改动：
  - 从 `receipt_date` 解析天粒度序号。
  - 归一化为 `[0,1]` 后挂到 `patient.time_days`（shape: `[N,1]`）。
  - 保留原 `date` 字段逻辑，兼容已有切分策略。

### 4) 在模型中接入“结构+时间”特征
- 修改文件：`models/base.py`
  - 新增 `Time2Vec` 实现。
  - `PreciseADR_RGCN/HGT` 新增辅助特征注入逻辑：
    - `patient_time` -> `Time2Vec + Linear` -> 与 `patient` 原输入相加
    - `drug_struct_feat` -> `Linear` -> 与 `drug` 原输入相加
  - `BasicModelWrapper.common_step()` 传入新字段。
  - `transfer_batch_to_device()` 新增 `struct_feat/time_days` 的搬运。
- 修改文件：`models/hetero_cl.py`
  - 同步接入上述辅助特征注入逻辑。
  - `ContrastiveWrapper.common_step()` 同步传入新字段。

### 5) 参数与配置接入

  - 新增参数：
    - `use_drug_struct`
    - `drug_encoder_type`
    - `drug_struct_dim`

    - `molformer_feat_path`
    - `use_time_feature`

- 修改文件：`config/all_HGT_config.yaml`

  - 默认开启结构与时间特征，并给出默认路径和维度。

### 6) 训练报告参数落盘增强
- 修改文件：`main.py`、`main_eval.py`
  - 将结构与时间相关参数写入报告，便于复现。
  - 在 `build_dataset_func()` 中回填 `args.drug_struct_dim`（从真实数据读取）。

## 设计取舍（为什么这样做）
1. 采用“在输入空间相加”的注入方式，而不是改全模型输入维度：  
   - 改动小、兼容现有 `in_lin` 与图卷积结构。  
   - 不破坏现有 patient/indication/SE 的输入流程。
2. 药物结构编码做成可切换模式：  
   - 便于后续 A/B 对比 `hybrid` 与 `molformer`。  
   - 在没有 RDKit 或没有 MolFormer 文件时也可运行。
3. 时间建模先走 Time2Vec：  
   - 复杂度低、可直接落地，适合当前工程状态。  
   - 后续若要升级时序图卷积，可在此基础上扩展。

## 本次增量修复（评估阶段 inference tensor 崩溃）

### 问题现象
- `main_eval.py` 在 `trainer.validate(...)` 阶段报错：
  `RuntimeError: Inplace update to inference tensor outside InferenceMode is not allowed`
- 报错栈位于 `torch_geometric` 的 `hetero_neighbor_sample`，属于评估采样阶段崩溃。

### 根因判断
- Lightning 默认评估走 `inference_mode`，与当前 PyG 邻居采样组合在该环境下存在兼容问题。
- 采样阶段发生原地更新时，命中了 inference tensor 限制。

### 修复改动
- 修改文件：`main.py`
  - 两处 `Trainer(...)` 增加 `inference_mode=False`：
    - 训练主流程的 Trainer
    - `main_predict` 的 Trainer
  - 说明：评估仍是无梯度执行（`no_grad`），只是关闭 inference tensor 机制，避免采样崩溃。

- 修改文件：`dataset/please.py`
  - 在 `dataloader(...)` 中，对入参 `mask` 增加 `mask = mask.clone()`。
  - 目的：每次构建 Loader 使用独立 mask，降低底层共享张量导致的原地更新冲突风险。

### 验证
- 已执行语法检查：
  - `python -m py_compile main.py dataset/please.py`
  - 结果：通过。

## 本次增量实现（本地 MolFormer + drug-drug 共现图）

### 1) 新增本地 MolFormer 特征生成脚本
- 新增文件：`new_data_in/build_molformer_features.py`
- 主要能力：
  - 固定读取 `new_data_in/refined_data/drugbank_id_smiles.csv`
  - 只接受本地 HuggingFace 格式模型目录，不走联网下载
  - 使用 `AutoTokenizer + AutoModel`
  - 对每个药物的 SMILES 做 `attention_mask` 加权 mean pooling
  - 输出宽表 CSV：
    - 第一列固定 `DrugBank ID`
    - 其余列固定为 `mf_0000`, `mf_0001`, ...`
  - 默认输出到 `new_data_in/refined_data/drugbank_molformer_features.csv`
- 实现细节：
  - 空 SMILES 不参与编码，但仍保留原顺序行，未编码位置写零向量
  - 保存前统一把 `NaN/Inf` 替换为 `0`
  - 脚本结束打印总数、成功数、失败数、输出维度和输出文件路径

### 2) 增强 `dataset/mol_features.py` 的 MolFormer 读取能力
- 修改文件：`dataset/mol_features.py`
- 主要改动：
  - `load_molformer_map()` 现在优先支持宽表 `DrugBank ID + mf_*`
  - 同时兼容旧格式 `embedding` 字符串列
  - `build_drug_struct_matrix()` 在 `molformer` 模式下：
    - 如果外部文件存在，直接使用文件真实维度
    - 不再强制按 `drug_struct_dim` 截断已有 MolFormer 向量
    - 某个药物缺失 MolFormer 时，单药回退到 `hybrid`
  - 返回统计补充：
    - `loaded_molformer_dim`
    - `missing_molformer_ids`

### 3) 在 `PLEASE` 图里显式加入 drug-drug 共现边
- 修改文件：`dataset/please.py`
- 主要改动：
  - 新增参数：
    - `use_drug_cooccur`
    - `drug_cooccur_min_count`
  - 在当前数据集 `all_pd` 内部统计每条病例的药物共现
  - 每条病例先对 `drugs` 去重，再做两两组合计数
  - 边类型新增为：`("drug", "d_d", "drug")`
  - 默认只保留共现次数 `>= 10` 的药物对
  - 同时把 `cooccur_count` 保存到边属性，方便后续分析
  - 打印关键统计：
    - `total_pairs_before_threshold`
    - `total_pairs_after_threshold`
    - `threshold`
    - `density_after_threshold`
- 同时升级了 processed 缓存命名：
  - 加入 MolFormer 文件标识、共现图开关和阈值
  - 避免旧缓存把新图结构覆盖掉

### 4) 补齐参数和默认配置
- 修改文件：`args.py`
  - 新增：
    - `use_drug_cooccur`
    - `drug_cooccur_min_count`
  - 默认正式切到：
    - `drug_encoder_type: molformer`
    - `molformer_feat_path: new_data_in/refined_data/drugbank_molformer_features.csv`
- 修改文件：
  - `config/all_HGT_config.yaml`
  - `config/age_HGT_config.yaml`
  - `config/gender_HGT_config.yaml`
- 默认值统一为：
  - `use_drug_struct: true`
  - `drug_encoder_type: molformer`
  - `molformer_feat_path: new_data_in/refined_data/drugbank_molformer_features.csv`
  - `use_drug_cooccur: true`
  - `drug_cooccur_min_count: 10`

### 5) 这次改动的设计原则
1. MolFormer 只做“离线产特征”，训练阶段只负责读取，不把大模型塞进训练流程。
2. 共现图严格按当前数据集统计，不把 `all` 的图偷渡到 `age/gender`。
3. 先只加新边类型和边属性，不去改 HGT/RGCN 的卷积接口，避免把改动面做大。
